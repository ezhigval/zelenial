# Документация ZELENIAL

Этот каталог специально сокращен до небольшого набора документов, у каждого из которых есть одна четкая зона ответственности.

## Порядок чтения

1. [../README.md](../README.md) — быстрый, самодостаточный обзор проекта.
2. [ARCHITECTURE.md](ARCHITECTURE.md) — главный технический источник истины: архитектура, стек, решения, структура, кодовые соглашения, примеры.
3. [SECURITY.md](SECURITY.md) — правила безопасности, приватности и обращения с данными.
4. [RULES.md](RULES.md) — процесс работы, правила репозитория, PR, review и взаимодействия с ИИ.
5. [ROADMAP.md](ROADMAP.md) — порядок реализации проекта по этапам.

## Принцип документации

- Один вопрос раскрывается подробно только в одном документе.
- В остальных документах допускается только краткая суть и ссылка на основной источник.
- `README.md` в корне должен оставаться самодостаточным, но не дублировать технические детали из архитектуры и правил.
