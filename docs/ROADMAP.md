# Roadmap ZELENIAL

Этот roadmap задает порядок работы над проектом: этапы, подэтапы, шаги и критерии завершения. Он описывает не идею продукта, а последовательность реальной реализации.

## Этап 0. База проекта и фиксация направления

- [x] Очистить репозиторий от старых экспериментальных наработок.
- [x] Зафиксировать новую модель: модульный монолит + гексагональная архитектура.
- [x] Перевести frontend на модель единого универсального веб-приложения.
- [x] Сократить документацию до компактной системы без дублей.
- [x] Подготовить новый каркас `backend/` и `frontend/`.

Критерий завершения:

- репозиторий чистый, структура целевая, документация согласована между собой

## Этап 1. Проектирование доменной модели и технического каркаса

### 1.1. Доменные границы

- [ ] Утвердить список блоков первого релиза.
- [ ] Описать зону ответственности каждого блока.
- [ ] Зафиксировать shared vocabulary и ключевые сущности.
- [ ] Определить границы между `Identity`, `Profile`, `Planning`, `Health`, `Finance`, `Learning`, `Reflection`, `Notifications`, `Assistant`.

### 1.2. Контракты и сценарии

- [ ] Описать первые `commands`, `queries` и `dto`.
- [ ] Выделить первичные `use cases`.
- [ ] Определить inbound/outbound ports.
- [ ] Согласовать минимальный набор API-контрактов версии `v1`.

### 1.3. Backend bootstrap

- [ ] Инициализировать базовый backend-проект на Go.
- [ ] Собрать composition root и конфигурацию.
- [ ] Подготовить слой HTTP adapter.
- [ ] Подготовить базовый persistence adapter.
- [ ] Добавить миграционный механизм и health endpoint.

### 1.4. Frontend bootstrap

- [ ] Инициализировать единый frontend-проект.
- [ ] Подготовить app shell, routing и базовый layout.
- [ ] Настроить PWA-режим.
- [ ] Подготовить заготовки под Capacitor и Tauri.
- [ ] Настроить базовый API client и обработку сессии.

Критерий завершения этапа:

- готов каркас backend и frontend без продуктовой глубины, но с рабочим skeleton под реализацию
