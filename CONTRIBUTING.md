# Contributing to ZELENIAL

Спасибо за интерес к проекту.

Перед любым вкладом обязательно прочитайте:

1. [README.md](README.md)
2. [docs/README.md](docs/README.md)
3. [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md)
4. [docs/RULES.md](docs/RULES.md)
5. [docs/SECURITY.md](docs/SECURITY.md), если изменение затрагивает данные, доступы или интеграции

## Базовый порядок работы

1. Определите одну четкую задачу.
2. Создайте отдельную ветку.
3. Вносите только логически связанные изменения.
4. Обновляйте документацию, если меняются архитектура, контракты или правила.
5. Перед PR проверьте, что изменение объяснимо и локально по ответственности.

## Что особенно важно

- соблюдать гексагональную архитектуру backend
- не плодить дублирующую документацию
- не смешивать бизнес-логику с инфраструктурой
- не добавлять временный мусор и артефакты сборки в git
- уважать процесс code review

## Pull Request checklist

- изменение соответствует [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md)
- изменение соответствует [docs/RULES.md](docs/RULES.md)
- описание PR объясняет цель, причину и область влияния
- выполнены релевантные проверки
- документация обновлена при необходимости

Если вы не уверены в направлении решения, сначала откройте обсуждение, а не тащите спорную архитектуру прямо в код.
