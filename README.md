# ZELENIAL

ZELENIAL — единое цифровое приложение для управления жизнью, здоровьем, планированием, финансами, обучением и личной рефлексией.

Репозиторий собран как чистая стартовая точка проекта: без старых экспериментальных наработок, с компактной документацией, понятной навигацией и каркасом под дальнейшую ручную разработку.

## Что зафиксировано в проекте сейчас

- Backend строится как моноприложение с гексагональной архитектурой.
- Frontend строится как единое веб-приложение для браузера, PWA, iOS, Android, macOS и Windows.
- Внутренние блоки системы независимы и взаимодействуют только через сущности, контракты, `use cases` и порты.
- Документация разделена по зонам ответственности без повторения одного и того же текста.
- Основная роль ИИ в проекте: reviewer, mentor и архитектурный помощник, а не основной автор продуктового кода.

## Ключевая идея

ZELENIAL должен помогать пользователю не в одной узкой задаче, а в ежедневном управлении всей системой жизни:

- личный кабинет, идентификация, безопасность и настройки
- планирование, календарь, задачи, привычки и напоминания
- здоровье, состояние, сон, стресс и самонаблюдение
- финансы, бюджет, регулярные платежи и контроль расходов
- обучение, развитие навыков, заметки и рефлексия
- интеллектуальные рекомендации и аккуратная персонализация

## Архитектурный вектор

### Backend

- Один монолит вместо раннего дробления на микросервисы.
- Четкие доменные границы внутри монолита.
- Гексагональная архитектура: доменное ядро не зависит от HTTP, БД, UI и внешних инструментов.
- Вся интеграция проходит через порты и адаптеры.

### Frontend

- Один кодбейс вместо отдельных приложений под каждую платформу.
- Базовый режим доставки: браузер и PWA.
- Мобильная упаковка: Capacitor.
- Десктопная упаковка: Tauri.

Подробная техническая модель описана в [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md).
